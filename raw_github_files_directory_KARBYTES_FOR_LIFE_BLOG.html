The following hyperlinks each lead to exactly one raw file which is part of the public GitHub repository named KARLINA_OBJECT. The purpose of this web page is to save karbytes time and energy when saving each raw file in that GitHub repository to the WayBack Machine (as a web page which can be publicly viewed on the WayBack Machine database). karbytes intends to routinely log into its Internet Archive user account in order to access the batch save feature of the WayBack Machine (which allows karbytes to save the web pages which are linked to one directory web page which displays outgoing hyperlinks to those web pages when karbytes enters the Uniform Resource Locator (URL) of that directory web page to the text field on the WayBack Machine save form and then clicks the “SAVE PAGE” button to begin saving that directory web page and the web pages which are referenced via hyperlinks on that directory web page).

Note that, for a directory web page which has more than a specific number of hyperlinks on it (i.e. “many” hyperlinks), some of (but not all of) the web pages which are referenced via hyperlinks on that directory web page are saved in a “random” order during a batch WayBack Machine save operation (and a batch save occurs as a result of selecting the “save outlinks” option on the WayBack Machine “Save Page Now” input form). A directory web page with “many” hyperlinks on it will need to be saved multiple times in order for each web page which is referenced by a hyperlink on that directory web page to be saved via batch save operation.

Note that the aforementioned GitHub repository (which is referenced on this web page using the green highlighted GitHub repository hyperlink in the yellow highlighted paragraph) can be saved to the WayBack Machine as a directory web page (using the “save outlinks” feature) such that the non raw files which are in that repository are saved as WayBack Machine web pages via batch save operation.

This web page exists on the website named Karbytes For Life Blog dot WordPress dot Com because raw files which are part of a GitHub repository are have no direct outgoing hyperlinks from the web page whose URL is the web page address of that GitHub repository.
